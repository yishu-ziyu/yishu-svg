# 产品需求文档：Aegis 智能 SVG 教学动画生成器

## 1. 产品定义与核心价值

**背景：**
在传统的数字学习与教学中，知识的传递往往受限于静态图文。文字抽象难懂，视频制作成本高昂且难以编辑。学习者和教育者缺乏一种**低成本、高效率、可交互**的方式来动态演示复杂的逻辑或变化过程。

**产品定义：**
Aegis "SVG 教学动画" 是一个基于 LLM 的即时渲染工具。它允许用户通过自然语言描述，一键生成带有动态效果的 SVG（可缩放矢量图形）动画。

**核心价值（解决的问题）：**

- **认知降维：** 将抽象的数学公式、物理定律或历史进程，转化为直观的动态视觉，降低理解门槛。
- **即时生产：** 替代传统的 Flash/AE 制作流程，将制作时间从"小时级"压缩至"秒级"。
- **无损交互：** 基于 SVG 代码生成，图像无限放大不失真，且代码本身可被再次编辑和复用。

---

## 2. 用户场景 (Use Cases)

### 场景一：数学/物理概念的动态推演

- **用户痛点：** 老师在讲"二次函数"时，静态图片无法展示参数变化如何影响图像开口和顶点。
- **Aegis 解决方案：** 用户输入"讲解二次函数的图像与顶点"，系统生成抛物线动画，动态标注顶点的移动轨迹。

### 场景二：复杂逻辑与结构的可视化

- **用户痛点：** 解释"神经网络"或"计算机架构"时，层级关系和数据流向难以用语言描述清楚。
- **Aegis 解决方案：** 用户输入"人工神经网络结构"，系统生成输入层、隐藏层、输出层的节点连接图，并用动态线条模拟数据传输过程。

### 场景三：人文知识的情境化与时间轴

- **用户痛点：** 学习历史事件或文学意象时，缺乏空间感和时间感。
- **Aegis 解决方案：** 用户输入古诗《静夜思》，系统生成动态意境配图，将离散的知识点串联起来。

## 3. 功能详细说明 (Functional Requirements)

### 3.1 交互入口与流程

- **入口：** 聊天框功能区，选择 `SVG 教学动画`
- **参数配置：**
  - **风格 (Style)：** 下拉菜单（`Manim风格`、`时间轴`、`极简风格`）
  - **教学动画要求 (Prompt)：** 文本输入框
  - **附件：** 支持上传参考图片（可选）

### 3.2 生成逻辑 (Generation Logic)

- **模型指令：** 将用户输入转化为 System Prompt，要求 LLM 输出符合 XML 标准的 `<svg>` 代码
- **关键技术点：**
  - **动画实现：** 使用 SVG 内置 `<animate>`, `<animateTransform>`, 或 CSS `@keyframes`
  - **坐标系控制：** 预设笛卡尔坐标系 `viewBox`
  - **排版美学：** 预设字体、配色方案和布局

### 3.3 渲染与结果页 (Rendering & Output)

- **代码预览：** 显示 SVG 源代码
- **画布渲染：** 直接渲染 SVG 动画
- **操作栏：**
  - `重制` - 重新生成
  - `下载` - 保存为 .svg / .mp4 / .gif
  - `全屏` - 独立窗口查看

## 5. 未来迭代方向

1. **交互式微调：** 通过自然语言修改动画（"把背景改成白色"）
2. **数据驱动：** 支持上传 Excel/CSV 生成动态图表
3. **组件复用：** 建立 SVG 资产库，减少幻觉风险
